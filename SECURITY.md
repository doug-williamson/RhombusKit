# Security Policy

## Supported versions

RhombusKit is released in **lockstep** — all four packages
(`@rhombuskit/tokens`, `@rhombuskit/theme-engine`, `@rhombuskit/material-preset`,
`@rhombuskit/core`) share one synced version. Security fixes land on the latest
stable line.

| Version | Supported |
|---|---|
| `1.x` (latest) | ✅ |
| `< 1.0` | ❌ |

A security fix ships as a patch across all four packages on one synced version,
the same way every release does.

## Reporting a vulnerability

**Please report security issues privately — do not open a public issue or
Discussion, and do not include exploit details in a public PR.**

Use GitHub's private vulnerability reporting:

➡️ **[Report a vulnerability](https://github.com/doug-williamson/RhombusKit/security/advisories/new)**

…or, if you prefer email: **douglasmwilliamson@gmail.com**.

Please include:

- the affected package(s) and version,
- a description of the issue and its impact,
- and steps to reproduce (a minimal proof-of-concept helps).

## What to expect

- An acknowledgement of your report, typically within a few days.
- An honest assessment of severity and a fix timeline.
- Credit for the disclosure in the release notes, if you'd like it (and only
  after a fix is available).

Thank you for helping keep RhombusKit and its users safe.
