# Motion Foundation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Promote motion in RhombusKit from ad-hoc per-component magic numbers to a tokenized, documented, accessibility-correct subsystem — then apply it to the one overlay (popover) that currently has no entrance animation.

**Architecture:** Motion durations and easing curves are theme-independent, so they live as **primitives** in `packages/tokens/src/spec/primitives.ts` (alongside `radius`/`fontFamily`) — *not* in the frozen semantic CONTRACT. The token generator flattens them to `:root` CSS custom properties (`--motion-duration-*`, `--motion-ease-*`). Components consume them via raw `var(--…)` exactly as they already consume `--radius-md`. Reduced-motion is already handled globally by `packages/tokens/src/styles/_reset.scss`, so no per-component reduced-motion code is added.

**Tech Stack:** Angular (standalone, OnPush, signals) · nx monorepo · Sass component styles · TypeScript token spec + `tsx` generator · Jest + jest-axe · Angular CDK Overlay (popover).

## Global Constraints

- **Tokens are generated.** Never hand-edit `packages/tokens/src/generated/*`. Edit `packages/tokens/src/spec/*`, then regenerate. The generator is `tools/generate-tokens.mjs`.
- **The semantic CONTRACT is frozen / append-only and is NOT touched by this work.** Motion lives in `primitives`, which is not under the `contract.snapshot.json` freeze. Do not add motion names to `packages/tokens/src/types.ts`.
- **`pnpm` is not on this shell's PATH.** Run every nx command through Corepack: `corepack pnpm exec nx <…>`. (nx itself spawns nested `pnpm run …`, which also needs Corepack.)
- **Component SCSS references tokens via raw `var(--…)` with no `@use`.** Keep that convention — do not introduce a shared SCSS mixin import into `packages/core/src/lib/**`.
- **Reduced-motion is handled once, globally**, by `@media (prefers-reduced-motion: reduce)` in `packages/tokens/src/styles/_reset.scss` (it zeroes `transition-duration`/`animation-duration` on every element). Do not add per-component `prefers-reduced-motion` blocks.
- **Do not run `nx build core` while `nx serve showcase` is running** (ng-packagr cleans `dist`, which wedges the showcase's global `@use '@rhombuskit/core/scss'`).
- **Inline `styles:` arrays must use `/* … */` comments, never `//`** (Sass `//` leaks into emitted CSS; `tools/verify-component-styles.mjs` enforces this on published packages).
- `.gitattributes` is `* text=auto` (LF-normalized on commit). After regenerating tokens, confirm the generated diff contains only motion lines; if Windows CRLF noise appears, `git add --renormalize` the generated dir before committing.

## The motion token scale (reference for every task below)

Added to `primitives.ts` as the `motion` group. Flattened CSS custom properties on the left; rationale on the right.

| Token | Value | Use |
|---|---|---|
| `--motion-duration-instant` | `0ms` | No-op baseline |
| `--motion-duration-fast` | `120ms` | Micro-feedback: hover tints, small bg/color shifts |
| `--motion-duration-base` | `150ms` | Default UI transitions: buttons, links, focus (matches the dominant existing value, so the migration is visually a no-op) |
| `--motion-duration-slow` | `240ms` | Larger/expressive moves: overlay entrances, sort sweeps |
| `--motion-ease-standard` | `cubic-bezier(0.4, 0, 0.2, 1)` | Default; Material-standard curve (already used by data-table sort) |
| `--motion-ease-emphasized` | `cubic-bezier(0.2, 0, 0, 1)` | Expressive, for prominent moves |
| `--motion-ease-decelerate` | `cubic-bezier(0, 0, 0.2, 1)` | **Enter**: fast→slow, settles to rest |
| `--motion-ease-accelerate` | `cubic-bezier(0.4, 0, 1, 1)` | **Exit**: slow→fast, accelerates away |

**Note on the migration:** existing sites use the CSS `ease` keyword and three durations (120/150/225ms). Tokenizing standardizes easing to `--motion-ease-standard` and normalizes 225ms→240ms. At ≤240ms these differences are imperceptible; consolidation is the intended improvement.

## Phase B — Popover entrance choreography

### Task 3: Add a tokenized CSS enter animation to the popover panel

The popover is the only overlay with **no** animation (dialog/menu/select/toast/tooltip inherit Material's). It's a custom CDK overlay we fully own. The panel is freshly attached on every open, so a CSS `animation` on the panel selector plays on each mount with **zero TypeScript change**.

**Critical constraint — enter only, no delayed teardown.** The popover specs assert the panel is removed *synchronously* on close (e.g. `rhombus-popover.component.spec.ts:81-89`, the Escape/backdrop/`rhombusPopoverClose`/destroy tests). Delaying `overlayRef.dispose()` for an exit animation would break ≥5 tests. This task animates entry only; `close()` stays immediate. (Exit animation is a deliberate follow-up — see "Explicitly out of scope".)

`@keyframes` is global here because the component uses `ViewEncapsulation.None`, so its `styleUrl` is injected globally and matches the panel inside `.cdk-overlay-container`.

```scss
// The panel renders in the CDK overlay container, so its styles are global and
// scoped under .cdk-overlay-container (mirrors how RhombusMenu styles its panel).
// Colours/shadows via contract tokens; radius via the radius primitive (--radius-lg).
.cdk-overlay-container .rhombus-popover__panel {
  background: var(--surface-0);
  color: var(--text-primary);
  border: 1px solid var(--border);
  border-radius: var(--radius-lg);
  box-shadow: var(--shadow-md);
  padding: 8px;
  min-width: 180px;
  font-family: var(--font-sans);

  // Entrance: subtle fade + lift + scale, played on each mount. Decelerate
  // easing settles the panel to rest. Origin-neutral so it reads correctly
  // whether the panel opens below or above the trigger. Exit is intentionally
  // instant (the trigger disposes the overlay synchronously). Honoring
  // prefers-reduced-motion is global (styles/_reset.scss zeroes the duration).
  animation: rhombus-popover-enter var(--motion-duration-base)

  &:focus {
    outline: none; // focus is trapped to the first focusable child via cdkTrapFocus
  }
}

@keyframes rhombus-popover-enter {
  from {
    opacity: 0;
    transform: translateY(-4px) scale(0.98);
  }
  to {
    opacity: 1;
    transform: none;
  }
}
```

```ts
import { ChangeDetectionStrategy, Component } from '@angular/core';
import { RouterLink } from '@angular/router';
import { tokens } from '@rhombuskit/tokens';

interface MotionToken {
  name: string; // e.g. "--motion-duration-fast"
  value: string; // e.g. "120ms"
}

const PRIM = tokens.primitives;

// Derived from the generated primitives so this reference tracks the source.
const DURATIONS: MotionToken[] = Object.keys(PRIM)
  .filter((k) => k.startsWith('motion-duration-'))
  .map((k) => ({ name: `--${k}`, value: PRIM[k as keyof typeof PRIM] }));

const EASINGS: MotionToken[] = Object.keys(PRIM)
  .filter((k) => k.startsWith('motion-ease-'))
  .map((k) => ({ name: `--${k}`, value: PRIM[k as keyof typeof PRIM] }));

@Component({
  selector: 'app-motion-page',
  standalone: true,
  imports: [RouterLink],
```
